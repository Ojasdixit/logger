## 📋 Step-by-Step Deployment

### Step 1: Push to GitHub

Choose your authentication method:

**Option A: GitHub CLI (Recommended)**
```bash
brew install gh
gh auth login
# Select HTTPS, authorize browser popup
git push origin main
```

**Option B: Personal Access Token**
```bash
# 1. Go to https://github.com/settings/tokens
# 2. Generate new token (classic)
# 3. Select: repo scope
# 4. Copy token
# 5. Run:
git push origin main
# Username: your-github-username
# Password: (paste token)
```

---

### Step 2: Setup Cloudinary (Free)

Your backend needs image storage credentials:

```bash
1. Sign up: https://cloudinary.com/users/register/free
2. Go to Dashboard
3. Copy:
   - Cloud Name
   - API Key
   - API Secret
4. Update backend/.env:
   CLOUDINARY_CLOUD_NAME=your_value
   CLOUDINARY_API_KEY=your_value
   CLOUDINARY_API_SECRET=your_value
5. Commit & push:
   git add backend/.env
   git commit -m "chore: add Cloudinary credentials"
   git push origin main
```

---

### Step 3: Deploy Backend to Render

Render handles deployment automatically from GitHub:

```bash
1. Go to https://render.com
2. Sign up with GitHub
3. Click "New +" → "Web Service"
4. Connect your GitHub repo: Ojasdixit/logger
5. Configure:
   - Root directory: backend
   - Build command: npm install
   - Start command: node src/index.js
   - Environment variables (from backend/.env):
     SUPABASE_URL=https://dmmfzdvhojihqtcxaxxo.supabase.co
     SUPABASE_SERVICE_KEY=eyJhbGciOiJ...
     CLOUDINARY_CLOUD_NAME=your_value
     CLOUDINARY_API_KEY=your_value
     CLOUDINARY_API_SECRET=your_value
     ADMIN_USERNAME=admin
     ADMIN_PASSWORD=monitor2026
     AGENT_API_KEY=agent-secret-key-2026
6. Click "Create Web Service"
7. Wait for deployment (2-3 minutes)
8. Copy the deployed URL: https://your-app-backend.onrender.com

✅ Backend is now LIVE!
```

---

### Step 4: Deploy Dashboard to Netlify

```bash
1. Go to https://netlify.com
2. Sign up with GitHub
3. Click "Add new site" → "Import an existing project"
4. Select your GitHub repo: Ojasdixit/logger
5. Configure:
   - Base directory: dashboard
   - Build command: npm run build
   - Publish directory: dist
   - Environment variables:
     VITE_API_URL=https://your-app-backend.onrender.com
6. Click "Deploy site"
7. Wait for deployment (1-2 minutes)
8. Your dashboard is live at: https://your-app.netlify.app

✅ Dashboard is now LIVE!
```

**⚠️ Update netlify.toml**
After you know your Render backend URL, update `dashboard/netlify.toml`:
```toml
# Line 7 and 10-12
VITE_API_URL = "https://your-app-backend.onrender.com"

# Line 19 (API redirect)
to = "https://your-app-backend.onrender.com/api/:splat"
```

Then commit & push:
```bash
git add dashboard/netlify.toml
git commit -m "chore: update backend URL for production"
git push origin main
```

Netlify auto-redeploys when you push!

## 🔧 If Something Breaks

### Backend deployment fails on Render
- Check Supabase URL and service key are correct
- Verify Cloudinary credentials
- Look at Render logs: Dashboard → your-service → Logs

### Dashboard shows "Cannot reach API"
- Check `VITE_API_URL` env var in Netlify settings
- Verify Render backend is actually running
- Check your browser's Network tab (should see API calls)

### Agent doesn't install
- Check installer script URLs point to your actual server
- Verify Node.js is accessible from terminal
- Run with verbose: `bash -x install-macos.sh`

### Employees show no activity
- Check backend logs for upload errors
- Verify agent config has correct API URL and key
- Check Cloudinary quota isn't exceeded (free tier: 25 images/month)

## 🔒 Security Checklist

Before going to production:

- [ ] Change `ADMIN_PASSWORD` (don't use monitor2026)
- [ ] Change `AGENT_API_KEY` (generate a random string)
- [ ] Use HTTPS everywhere (automatic with Render/Netlify)
- [ ] Rotate Cloudinary API credentials periodically
- [ ] Monitor Render logs for errors
- [ ] Don't commit `.env` files with real credentials
- [ ] Consider IP-restricting admin panel later
